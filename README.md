## 各フォルダ説明

### /admin
お知らせ・トップカルーセル・通報された作品を非表示・新規タグやカテゴリ作成等ができるページが入っています。<br>
管理権を付与されたアカウントのみでアクセスできます。

### /api
各ページから呼び出す処理が入っています。<br>
ほぼClaudeCodeが作ったものです。

### /cart
購入処理が個別ディレクトリになりindex.phpしか入ってないので今度cart.phpにリネームして外に出します。

### /category
カテゴリごとに作品を表示する個別ページを作る予定でしたが見栄えがするだけの作品を登録するのが大変なので後回しになっています。

### /common
/common/batchには指定した日時に/userdata/temp内のファイル（ユーザー・作品登録中に画像が一時保存される場所）を削除するcron用のphpが入っています。<br>
common/componentsはヘッダー等多くのページで利用するパーツをコンポーネント化して入れています。<br>
/common直下のphpファイルはログイン状態のチェック等に使うものです。

### /docs
TodoやDB内テーブルのカラム設定等メモを入れるフォルダです。

### /images
管理者がアップロードするサイトの普遍的な部分に使う画像フォルダです。

### /login
ログインやユーザー情報登録→仮登録→本登録URL等を入れています。

### /mypage
ユーザー個別のアカウントメニュー周りのページを置いています。

### /news
お知らせとお知らせ一覧のフォルダです。

### /other
ガイドラインのみが入っています。<br>
実サービスであれば利用規約や特定商取引法に基づく表記なんかも入る所です。

### /purchase
購入処理周りのページが入っています。

### /works
作品詳細を表示するページが入っています。


## ソースをDLしてローカル環境で動かす場合

### 依存パッケージのインストール<br>
→vendorフォルダはgit管理対象外なので、ソースコードのDL後にstallフォルダ内をターミナルで開き下記コマンドを実行してください。<br>
composer install

### .envファイルの構造例
→.env.exampleにあります。

### データベーススキーマ
→docs/schema.sql内のSQL文を作成した空のデータベースを選択した状態で実行すると全12テーブルが再現されます。

### userdataフォルダの作成
userdata内のファイルは.gitignoreしているのでソースコードのDL後にstallフォルダ内にuserdataフォルダを、
userdataフォルダ内には icon,temp,thumbnail,works-image,works
上記5つのフォルダを追加で作成してください。

あとはXAMPP（PHPのローカル開発環境を作るソフト）にデフォルトで付属していたので自分では入れた覚えがないんですが

 PHP拡張の確認 — pdo_mysql・mbstring・fileinfo・curl・opensslが必要（Stripe SDKがcurl/opensslを使用、download.phpがfileinfoを使用）。XAMPP標準なら通常有効
 Apacheのmod_rewriteが有効 — .htaccessで拡張子省略アクセスをしているため
 
以上の拡張が有効になっているらしいです
